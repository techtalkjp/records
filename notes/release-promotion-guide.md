# リリース告知・プロモーション ガイド

## タイムラインと各プラットフォームの使い分け

### 3/25 のスケジュール例

```
0:00  Spotify / Apple Music 公開（DistroKid経由）
      → DistroKid HyperFollow リンクが有効に
12:00 YouTube プレミア公開
      → 同時に X に投稿
```

### プラットフォーム別の役割

| プラットフォーム | 役割 | コンテンツ |
|---|---|---|
| **X (Twitter)** | 速報・拡散・会話 | テキスト + 動画クリップ or カバーアート画像 |
| **YouTube** | フル視聴体験 | リリックビデオ（プレミア公開） |
| **Spotify / Apple Music** | ストリーミング | 楽曲本体 |
| **DistroKid HyperFollow** | 誘導ハブ | 全プラットフォームへのリンク集約 |

### 投稿のゴールデンタイム
- X: 20〜22時がゴールデンタイム（社会人・学生がひと段落つく時間）
- ただし昼12時のYouTubeプレミアに合わせるのもあり（昼休み需要）

## リンク戦略

### DistroKid HyperFollow（無料）
- リリースごとに自動生成されるランディングページ
- Spotify / Apple Music / YouTube Music etc. へのリンクを集約
- SNSリンクも追加可能
- **告知投稿にはこのURLを貼るのが基本**

### スマートリンクツール（必要に応じて）
- **Linkfire** — メジャーレーベルが使用。高機能だが有料
- **Smarturl** — 無料だがやや使いにくい
- HyperFollowで十分な場合が多い

## 投稿パターン

### パターン1: 先行告知（リリース前日〜当日朝）

```
🎵 明日 3/25 リリース

新トラック「ターミナルの誇り」

Spotify / Apple Music: 0:00〜
YouTube プレミア: 12:00〜

[HyperFollow URL]

#日本語ラップ #AIrhymes #ClaudeCode
```

### パターン2: YouTube プレミア告知（当日昼）

```
新トラック「ターミナルの誇り」
YouTube プレミア公開 🎬

[YouTube URL]

Spotify / Apple Musicでも配信中
[HyperFollow URL]

#日本語ラップ #AIrhymes #ClaudeCode
```

動画を直接添付する場合はYouTube URLは不要（X上で再生できるため）

### パターン3: ストリーミング誘導（当日夜のゴールデンタイム）

```
「ターミナルの誇り」配信中

Spotify / Apple Music / YouTube Music
👉 [HyperFollow URL]

#日本語ラップ #AIrhymes #ClaudeCode
```

## X投稿のベストプラクティス

- **動画添付 > 画像添付 > テキストのみ** の順でエンゲージメント高い
- TikTokやReelsをそのままX/Blueskyに転載しない（パフォーマンスが悪い）
- 投稿を3-7日間隔で分散させると、繰り返し目に入って効果的
- ハッシュタグは2-3個まで。多すぎるとスパム感
- カバーアート画像の添付は、曲の「顔」を覚えてもらうのに有効

## DistroKid HyperFollow の設定

1. DistroKid管理画面からHyperFollowページを確認
2. 表示するサービスを選択・並び替え（Spotify, Apple Music, YouTube Music を上位に）
3. SNSリンクを追加
4. カスタムURLを設定（可能なら）

## Spotify For Artists

- リリース7日以上前にSpotify For Artistsで楽曲をピッチ → Release Radarに載る可能性
- プロフィール画像、ヘッダー、バイオを更新

## このプロジェクトでの告知スタイル

時事ネタやエンジニアコミュニティの文脈を活かす：
- 前回の「行ってこい」は Cursor Composer 2 ダウンに合わせて投稿 → 反応良かった
- テック業界の出来事に絡めるとエンジニア層に刺さる
- ただし曲自体は普遍的な内容なので、テック文脈に限定しすぎない
