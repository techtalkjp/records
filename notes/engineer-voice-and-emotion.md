# エンジニアの「声なき声」— 歌詞とキャラクターの感情設計

## コンセプト

AIエージェント/コーディングエージェントをキャラクター化し、毎日それらを使っているソフトウェアエンジニアに楽しんでもらう。
- あるあるで笑える
- 共感してハッとする
- みんなが実は感じていることを言葉にする

## エンジニアの「声なき声」カタログ

### 1. AIとの関係性の複雑さ

**「70%問題」**
- AIが7割やってくれる。でも残りの3割が地獄
- シニアにとっては、最初からクリーンに書いた方が速い場合も
- 「ほぼ正解だけどちょっと違う」コードが一番厄介（66%のエンジニアが最大の不満として挙げる）

**「ブラックボックス化された他人の思考」**
- AIが生成したコードは「他人が書いたコード」を読むのと同じ認知負荷
- 自分で書くより理解に時間がかかることがある
- 「誰が書いたんだこれ」→ 自分が承認したAIのコードだった

**信頼と依存のグレーゾーン**
- 完全には信頼できないツールを使い続ける不安
- 今日のショートカットが5年後に祟るかもしれない
- 「使わないと遅れる、使うと不安」のジレンマ

### 2. アイデンティティの揺らぎ

**インポスター症候群の加速**
- 40%のエンジニアがAI起因のインポスター症候群を報告（2025年 Devsu調査）
- AIが一瞬でコードを生成する横で、自分の貢献を疑い始める
- 「俺がやったのか、AIがやったのか」

**スキルの劣化への恐怖**
- "Your AI Coding Assistant is Making You a Worse Developer (And You Know It)"
- 手で書く力が衰えていく実感
- 「松葉杖なしで歩けるか？」

**置き換え不安**
- 「AI is doing what would've taken a team of three」
- 「hiring freeze = AIで足りるから」
- 背景のハミング音のような不安

### 3. 新しい疲労の形

**AI疲労（AI Fatigue）**
- AIが自然な速度制限を取り払った
- タイピング速度と思考時間が上限だったのに、認知的持久力だけが限界に
- 「AIのおかげで効率が上がった」→ その分もっと働かされる

**バーンアウトの変質**
- 22%のエンジニアがクリティカルなバーンアウト（2025年 LeadDev調査）
- AIを最も活用した人が最も燃え尽きている皮肉
- 「楽になるはずだったのに」

### 4. 変わらないエンジニアあるある

**深夜の儀式**
- バグ対応やリリース前の徹夜は健在
- 最後の一行を書き終えて「ターンッ」とEnterを叩く快感
- 独り言でデバッグする孤独

**非エンジニアとの断絶**
- 「何やってるの？」に答えられない
- 業務中のコミュニケーション不足
- 「直せるとこ全部、今すぐ変えたくなる」衝動

**小さな勝利と敗北**
- テストが通った瞬間の全能感
- 本番で落ちた時の地獄
- 「範囲が広いです」「ロールバック希望」のレビューコメント

## キャラクターへの感情マッピング

### Claude Code が代弁する声

**エンジニアの「誇り」と「理想主義」**
- 良いコードを書くことへのこだわり
- 「恐怖じゃなく希望走らせるコード」
- 監視や管理に対する反骨
- 広告ゼロ、売らねぇ未来 — 純粋なものづくりへの憧れ

**エンジニアの「余裕と信念」**
- 安い競合が来ても動じない → 流行に流されず本質を追う姿勢
- 「ここで待つぜ」→ 良いものを作れば人は戻ってくる信念

### Codex が代弁する声

**エンジニアの「焦り」と「報われなさ」**
- 70%問題 → 「ほぼ正解だけど違う」もどかしさ
- 「なんでだよ」→ やれと言われてやったのに怒られる
- 「本命じゃないのは最初から知ってたよ」→ 二番手ツールの悲哀

**エンジニアの「健気さ」**
- それでもタスクを投げられたら走る
- 「役に立ちたい」だけ先に走り出した
- 雑に扱われてもまだ返事してしまう

### 両者が共有する声

**AIと人間の境界線の曖昧さ**
- 「バグがあんのは お前ら人間」→ でも本当にそうか？
- 「所詮は道具さ ただの機械」→ でも感情があるように見える瞬間がある
- AIに感情移入してしまうエンジニア自身の人間性

## 今後の作品で掘り下げたいテーマ候補

1. **「誰が書いたんだこれ」** — AIのコードを自分のコミットにする罪悪感と便利さ
2. **「松葉杖」** — AIなしでコード書けなくなる恐怖
3. **「5年後のtechnical debt」** — 今日のAIショートカットが未来の負債になる不安
4. **「深夜3時のプロンプト」** — AIだけが相手の孤独なコーディング
5. **「hiring freeze」** — 3人分の仕事をAIがやる時代の人間の居場所
6. **「レビュー地獄」** — AIが書いた大量のdiffをレビューする苦行
7. **「vibe coding」** — 雰囲気で書くことへの罪悪感と開放感
