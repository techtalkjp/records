# Nano Banana 2 顔一貫性テクニック

出典: https://x.com/suthio_/status/2035307361804693832

## 核心の問題

「同じキャラクターの顔を、異なるポーズ・衣装・シチュエーションで再現できない」= ガチャ問題

顔は目の大きさが0.5mm変わるだけで「別人」に見える。人間の顔認識能力は極めて精密。

## Level 1: プロンプト設計

### ベースプロンプトを固定し、絶対に改変しない

キャラクターの身体的特徴を定義する「ベースプロンプト」を一度決めたら一切変更しない。

### やってはいけないこと
- `CRITICAL: MAINTAIN EXACT SAME FACE` のような強調指示 → 逆効果。年齢の幼化や特徴の歪み
- `Anti-AI: slight film grain, natural skin pores` のようなリアリティ強化 → 顔のテクスチャが不自然に

### 特徴記述は「具体的かつ測定可能」に

「別の人間がこの記述だけで同じ人物を描けるか」というレベルの具体性。

例:
```
Hair: dark brown semi-long hair reaching just past shoulders, natural straight with soft ends, see-through bangs
Eyes: large gentle dark brown eyes, natural double eyelids, long soft lashes
Face: soft oval face, smooth clear fair skin, small straight nose
```

## Level 2: 構図とアスペクト比

- 顔が大きく映る構図を優先（全身ショットは一貫性が落ちる）
- 三面図は個別生成して後から合成（一枚に複数アングルは別人化する）
- 9:16（縦長）→ AIが全身を入れようとする
- **3:4 → バストアップが安定**

## Level 3: マルチターン会話（最重要）

### 単発生成の限界
プロンプトをどれだけ工夫しても確率的な生成。参照画像を渡しても限界がある。

### マルチターンが構造的に解決する

Turn 1: ベースキャラクター生成（ベースプロンプト + 正面ポートレート指示）
Turn 2: Turn 1の応答（テキスト + 生成画像）を含めて、次の指示を送る

```json
{
  "contents": [
    {"role": "user", "parts": [{"text": "<ベースプロンプト>"}]},
    {"role": "model", "parts": [
      {"text": "<Turn 1の応答テキスト>"},
      {"inlineData": {"mimeType": "image/png", "data": "<Turn 1の画像base64>"}}
    ]},
    {"role": "user", "parts": [{"text": "Same person, identical face... 次の指示"}]}
  ]
}
```

### 5ターンの壁
- 5ターンを超えると初期の顔の特徴が薄れる
- **対策: セッション分割** — 各セッションの最初に「ベースショット」を再生成してアンカーにする

### 派生ターンのプロンプト設計（3要素必須）

1. **同一人物の明示**: Same person, identical face, same hairstyle.
2. **固有特徴のリマインド**: 特徴的な要素を2-3個繰り返す
3. **変更点の指示**: 変えたい要素のみ記述

### 「一度に大きく変えない」が鉄則
アングルだけ変える、衣装だけ変える。段階的に変化させると安定。

## Level 4: 動画生成への応用

参照画像の選び方:
- 2-3枚が最適（正面 + 斜め + 横顔）
- ニュートラルな条件（極端なライティング避ける）
- 1K解像度以上

## まとめ

1. シンプルなベースプロンプト（複雑にするほど不安定）
2. 構図とアスペクト比の戦略
3. マルチターン会話で構造的に一貫性を担保
4. セッション分割で長期的な一貫性を維持

**「プロンプトを複雑にするほど顔が安定する」という直感は間違い。**
シンプルなベースプロンプト + マルチターンの構造的アプローチが最も効果的。
